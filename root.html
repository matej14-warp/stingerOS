02 hardware_flashing

estimated time: 5 minutes

> prepare media

grab a high-quality usb flash disk. low-quality drives will fail you. we recommend using 'dd' for the pure approach or 'etcher' if you need a gui.

> flash using dd (unix/macos)

identify your drive first. don't wipe your primary disk. check twice.

# find your device (e.g., /dev/sdb or /dev/disk4)
lsblk 
# write the iso to the disk
sudo dd if=stinger.iso of=/dev/sdX bs=4M status=progress oflag=sync

> flash using windows

yeah yeah, you’re on windows. fine. don’t screw this up. use rufus or balena etcher. 

# rufus method (recommended)
1. plug in your usb drive
2. open rufus
3. select your usb device (double check it's not your main disk)
4. select stinger.iso
5. partition scheme: MBR
6. target system: BIOS (or UEFI-CSM)
7. click START and wait

# balena etcher method (simpler)
1. open etcher
2. select stinger.iso
3. select target usb
4. click flash

* rufus may ask about iso/dd mode — pick dd mode.
* windows might hide drives — triple check before flashing.
* if you wipe your main disk, that's on you.

> boot real hardware

plug the flash disk into your target x86 machine. enter bios/uefi and select the usb device. ensure legacy boot is enabled if needed. don't fuck it up.

* stinger os requires a 32-bit (i686) compatible cpu.
* minimal 128mb ram recommended for stable operation.
